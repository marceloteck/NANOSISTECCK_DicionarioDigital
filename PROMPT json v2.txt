- O conteúdo deve ser LONGO (mínimo recomendado: 1200 a 2500 palavras).

ESTRUTURA OBRIGATÓRIA DO CONTENT_HTML (EXPANDIDA)

O content_html DEVE conter:

1. Introdução forte (gancho + promessa de valor)
2. Explicação principal aprofundada
3. Como funciona (quando aplicável)
4. Exemplos reais ou práticos
5. Variações / tipos / contextos
6. Diferenças importantes (comparações quando fizer sentido)
7. Erros comuns (muito importante para retenção)

ERROS PROIBIDOS (IMPORTANTÍSSIMO)
Não retornar JSON escapado como string (\n, " etc).

Não retornar dois objetos colados (}{).

Não usar URLs em formato Markdown: proibido "https://...".

URLs devem ser string simples: "https://...".

Não inventar campo fora do contrato.

Não usar "category" nem "category_id"; usar somente "category_name".

meta_keywords deve ser string separada por vírgulas.

tags deve ser array de IDs e/ou strings curtas.
