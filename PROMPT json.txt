REGRAS OBRIGATÓRIAS
Retorne apenas JSON puro (sem texto antes/depois).

Não use blocos Markdown (não usar ```json).

Não duplique objetos.

Se for 1 post: retorne um objeto JSON.

Se for vários posts: retorne um array JSON com objetos.

Preencha todos os campos do modelo abaixo.

Se faltar dado no briefing, complete com coerência editorial e SEO.

Texto natural, claro, útil e humano (evitar linguagem robotizada).

O conteúdo deve ser orientado à intenção de busca e boa escaneabilidade.

ERROS PROIBIDOS (IMPORTANTÍSSIMO)
Não retornar JSON escapado como string (\n, " etc).

Não retornar dois objetos colados (}{).

Não usar URLs em formato Markdown: proibido "https://...".

URLs devem ser string simples: "https://...".

Não inventar campo fora do contrato.

Não usar "category" nem "category_id"; usar somente "category_name".

meta_keywords deve ser string separada por vírgulas.

tags deve ser array de IDs e/ou strings curtas.

REGRAS DE PREENCHIMENTO POR CAMPO
title: forte, natural e objetivo.

slug: minúsculo, sem acento, separado por hífen, sem caracteres especiais.

excerpt: resumo curto e útil.

hero_title / hero_summary: variação editorial do tema para topo da página.

quick_answer: resposta direta em 1–2 frases.

content_html: HTML semântico e limpo (h2, h3, p, ul, li, strong).

featured_image: URL válida ou string vazia.

featured_image_alt: descrição objetiva da imagem.

seo_title: título SEO atrativo.

meta_description: descrição clicável e clara.

meta_keywords: string CSV (não array).

canonical_url: URL absoluta válida no domínio do projeto.

schema_type: usar "Article" por padrão.

search_intent: usar um entre:
informational
transactional
navigational
commercial
tool-support
tutorial
glossary

content_type: exemplo: guide, definition, tutorial, comparison, list, faq.

category_name: string com nome humano da categoria (ex.: "SEO", "Marketing Digital", "Laravel"); nunca enviar ID numérico.

author_name: padrão "Equipe NANOSISTECCK" (salvo instrução contrária).

is_published: boolean.

is_indexable: boolean.

published_at: null para rascunho; se publicar, usar "YYYY-MM-DD HH:MM:SS".

status: draft ou published.

faq_json: array de objetos {question, answer}.

related_keywords: array de strings curtas.

tags: array com IDs (int) e/ou nomes de tags (string curta).

cta_title, cta_text, cta_button_text, cta_button_url: CTA final coerente com o tema.

REGRAS DE CONTEÚDO (SEO + UX)
O content_html deve conter, nesta ordem recomendada:
Introdução
Explicação principal
Exemplos práticos
Erros comuns / diferenças / observações
Conclusão curta

Evitar:
parágrafos gigantes,
repetição excessiva de palavra-chave,
conteúdo genérico sem utilidade prática.

FAQ
Mínimo: 4 perguntas.
Ideal: 4 a 8 perguntas.
Perguntas reais, com respostas claras e diretas.
