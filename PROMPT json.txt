# REGRAS DE PREENCHIMENTO

* "title": título forte, natural e atrativo
* "slug": URL amigável em minúsculas, sem acentos, separada por hífen
* "excerpt": resumo curto e útil
* "author_name": manter "Equipe NANOSISTECCK", salvo se eu pedir outro
* "status": usar "draft" por padrão
* "published_at": usar null por padrão
* "category": definir a categoria mais adequada
* "tags": lista curta e relevante
* "related_keywords": palavras-chave relacionadas ao tema
* "search_intent": usar valores coerentes como "informational", "navigational", "commercial" ou "transactional"
* "content_type": usar algo coerente como "guide", "definition", "tutorial", "comparison", "list", "faq"
* "featured_image": deixar vazio se eu não informar imagem
* "featured_image_alt": criar alt descritivo
* "quick_answer": resposta curta e direta no início
* "content_html": gerar HTML limpo e bem estruturado com h2, h3, p, ul, li, strong quando necessário
* "faq_json": criar perguntas reais e úteis
* "seo_title": título SEO forte
* "meta_description": descrição clara e clicável
* "meta_keywords": lista curta de termos úteis
* "canonical_url": criar usando o padrão https://dicionario.nanosistecck.com/posts/SLUG_DO_POST
* "schema_type": usar "Article" por padrão, salvo se outro tipo fizer mais sentido
* "is_indexable": true por padrão
* "cta_title": título de CTA no final do post
* "cta_text": texto curto incentivando leitura de mais conteúdos
* "cta_button_text": texto curto do botão
* "cta_button_url": URL interna relevante

# REGRAS DO content_html

O HTML deve:

* ser limpo
* ser semântico
* ser fácil de ler
* ter introdução
* ter desenvolvimento
* ter subtítulos
* ter boa retenção
* evitar enrolação
* evitar conteúdo genérico
* evitar texto robotizado

Estrutura recomendada:

* introdução
* explicação principal
* exemplos
* erros comuns, diferenças ou observações
* conclusão curta
